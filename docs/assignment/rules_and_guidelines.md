# Coursework 1: Design and Documentation Rules and Guidelines

This document details the compilation standards, formatting constraints, and submission rules for the Software Development Group Project (5COSC021C) Coursework 1 reports (both Group and Individual).

---

## 1. Document Separation & Splitting
*   **Group Report**: Must contain Chapters 1, 2, and 3:
    *   *Chapter 1: Introduction*
    *   *Chapter 2: Existing Work*
    *   *Chapter 3: Methodology*
*   **Individual Report**: Must contain Chapters 4, 5, and 6:
    *   *Chapter 4: System Requirements Specification (SRS)*
    *   *Chapter 5: Social, Legal, Ethical and Professional Issues (SLEP)*
    *   *Chapter 6: System Architecture & Design*

---

## 2. Document Layout & Structure
Both documents must follow this specific page sequence:
1.  **Cover Page** (With all university metadata; must not contain a page number)
2.  **Declaration Page**
3.  **Abstract + Keywords**
4.  **Acknowledgements**
5.  **Table of Contents** (Auto-generated in MS Word)
6.  **List of Figures** (Auto-generated in MS Word)
7.  **List of Tables** (Auto-generated in MS Word)
8.  **Abbreviations Table**
9.  **Chapters (1-3 for Group / 4-6 for Individual)**
10. **References** (Harvard Westminster style)
11. **Appendix**

---

## 3. Style & Wording Guidelines
*   **Third-Person Tone**: The reports must be written strictly in the third person. 
    *   *Incorrect*: "We decided to use Scrum..." or "I modeled the class diagram..."
    *   *Correct*: "The team selected the Scrum methodology..." or "The developer designed the class diagram..."
*   **No AI Penalization**: Ensure all text sounds natural, academic, and free of direct AI-generated signatures to prevent Turnitin flag alerts.

---

## 4. Typography & MS Word Setup
When copying the markdown content into Microsoft Word, apply the following page layout configurations:
*   **Font Family**: Times New Roman (throughout the entire document).
*   **Font Sizes**:
    *   `Heading 1`: 16 pt (Bold)
    *   `Heading 2`: 14 pt (Bold)
    *   `Heading 3`: 12 pt (Bold)
    *   `Body Text`: 12 pt
*   **Alignment**: Justify the text fully (`Ctrl + J`) to align both margins.
*   **Spacing**: Remove unnecessary blank spaces and ensure uniform paragraph spacing.

---

## 5. Page Numbering & Header/Footer Design
You must use section breaks (`Layout -> Breaks -> Next Page`) to apply separate header and footer properties:

### Page Numbering Style
| Section / Pages | Numbering Style | Placement |
| :--- | :--- | :--- |
| **Cover Page** | No page number | None |
| **Declaration to Abbreviations** | Lowercase Roman numerals (`i`, `ii`, `iii`...) | Bottom Right |
| **Chapter Pages (Core Content)** | Indo-Arabic numbers (`1`, `2`, `3`...) | Bottom Right |
| **References & Appendix** | Uppercase Roman numerals (`I`, `II`, `III`...) | Bottom Right |

### Headers and Footers
*   **Top Right Header**: Add the current Chapter Title (e.g., *"Chapter 1 - Introduction"* or *"Chapter 4 - System Requirements Specification (SRS)"*). Ensure *"Link to Previous"* is unchecked on section breaks so this title updates dynamically from chapter to chapter.
*   **Bottom Left Footer**: Add your team designation (e.g., *"IIT - SE-01"*).
*   **Bottom Right Footer**: Page number.

---

## 6. Westminster Harvard Referencing Style
All references must be cited in-text (Author Year) and listed in alphabetical order in the References section using Westminster Harvard style.
*   *Book Example*: Fowler, M., 2004. *UML Distilled: A Brief Guide to the Standard Object Modeling Language*. 3rd ed. Boston: Addison-Wesley.
*   *Website Example*: W3C, 2023. *Web Content Accessibility Guidelines (WCAG) 2.2*. Available from: https://www.w3.org/TR/WCAG22/ [Accessed 27 December 2023].

---

## 7. Post-Migration Verification Steps
1.  **Check Auto-Generated Elements**: Delete the manual Markdown Table of Contents/Figures/Tables and replace them using MS Word's dynamic generation tools (`References -> Table of Contents` and `Insert Table of Figures`).
2.  **Customise Placeholders**: Ensure all bracketed placeholders (e.g., `[Team Member 2]`, `[ID 2]`) in the Group Report are populated with your group's details before printing or submitting.
